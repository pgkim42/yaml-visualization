# YAML Visualizer v2 - 다중 시각화 기능 요구사항

## 1. 개요

### 1.1 기능명
다중 시각화 뷰 (Multi-Visualization Views)

### 1.2 목적
YAML 데이터를 다양한 시각화 형태로 표현하여 사용자가 데이터 구조를 더 직관적으로 이해할 수 있도록 지원

### 1.3 배경
- 기존 트리 뷰만으로는 복잡한 계층 구조 파악에 한계
- 데이터 비중/크기를 시각적으로 비교하기 어려움
- 전체 구조를 한눈에 파악하기 어려움

---

## 2. 시각화 유형

### 2.1 트리 뷰 (기존)
- 계층적 접기/펼치기 구조
- 상세 키-값 확인에 적합
- 검색 및 네비게이션

### 2.2 트리맵 (Treemap)
- 중첩 사각형으로 계층 표현
- 노드 크기로 데이터 비중 표현
- 전체 구조 비교에 적합

### 2.3 선버스트 (Sunburst)
- 동심원 형태 계층 표현
- 중심에서 바깥으로 깊이 증가
- 비율 및 구조 파악에 적합

### 2.4 마인드맵 (Mindmap)
- 노드-엣지 연결 다이어그램
- 자유로운 드래그/줌/팬
- 관계 파악 및 탐색에 적합

---

## 3. 기능 요구사항

### 3.1 뷰 전환
- 탭 형태의 뷰 선택 UI
- 즉시 전환 (데이터 재파싱 없음)
- 현재 뷰 상태 표시

### 3.2 상호작용
| 기능 | 트리 | 트리맵 | 선버스트 | 마인드맵 |
|------|------|--------|----------|----------|
| 노드 클릭 선택 | O | O | O | O |
| 검색 하이라이트 | O | O | O | O |
| 접기/펼치기 | O | - | - | - |
| 드래그 이동 | - | - | - | O |
| 줌/팬 | - | - | - | O |

### 3.3 동기화
- 모든 뷰에서 selectedPath 공유
- 검색 결과 모든 뷰에서 하이라이트
- 에디터 ↔ 뷰어 양방향 동기화 유지

### 3.4 테마
- 라이트/다크 모드 지원
- 노드 타입별 색상 일관성 유지

---

## 4. UI/UX 요구사항

### 4.1 뷰어 레이아웃
```
+------------------------------------------+
|  [트리] [트리맵] [선버스트] [마인드맵]     |  <- 탭
+------------------------------------------+
|                                          |
|           시각화 컨텐츠                   |
|                                          |
+------------------------------------------+
```

### 4.2 탭 디자인
- 아이콘 + 텍스트 조합
- 현재 선택 탭 강조
- 호버 효과

### 4.3 반응형
- 컨테이너 크기에 맞게 시각화 자동 조절
- 최소 너비/높이 보장

---

## 5. 기술 요구사항

### 5.1 라이브러리
- **Nivo**: 트리맵, 선버스트
- **React Flow**: 마인드맵

### 5.2 성능
- 동적 import로 번들 최적화
- useMemo로 데이터 변환 캐싱
- 대용량 데이터 처리 (1000+ 노드)

### 5.3 접근성
- 키보드 네비게이션
- 적절한 색상 대비

---

## 6. 성공 지표

- 4가지 시각화 뷰 정상 동작
- 뷰 간 상태 동기화 유지
- 다크모드 일관성
- 빌드 에러 없음
